Introduction to Data Visualization

Principles · Importance · Python Libraries Overview

What is Data Visualization?

Data Visualization is the graphical representation of information and data using visual elements like charts, graphs, maps, and dashboards to make data easier to understand, interpret, and communicate insights from.

Definition
Data visualization is the process of translating large datasets and metrics into charts, graphs, and other visuals that are easier to process and understand. It uses the human visual cortex's pattern-recognition ability to find insights in data quickly.

Importance in Different Areas

📊 Data Analytics

Quickly identifies trends, outliers, and patterns in raw data. Speeds up the exploratory data analysis (EDA) process. Communicates findings to non-technical stakeholders effectively.

🧠 Data Science

Validates model outputs and feature importance. Visualizes data distributions before ML training. Monitors model performance through dashboards and metrics tracking.

💼 Business Analytics

Enables data-driven decision making at all levels. KPI dashboards give real-time business health overview. Uncovers business opportunities and risks through trend analysis.

Benefits of Data Visualization

  • Faster Comprehension: Human brain processes visuals 60,000× faster than text
  • Pattern Recognition: Easily spot trends, correlations, and outliers
  • Better Decision Making: Evidence-based decisions from clear visual data
  • Storytelling: Data stories are more persuasive and memorable
  • Accessibility: Complex data becomes understandable to all stakeholders
  • Exploration: Interactive dashboards allow self-service data exploration
  • Error Detection: Anomalies and dirty data become obvious visually

PYTHON · SCATTER PLOT — ADVERTISING vs SALESQB Q4
import matplotlib.pyplot as plt
import numpy as np

np.random.seed(42)
ad_budget = np.random.uniform(1000, 10000, 50)
sales = 5 * ad_budget + np.random.normal(0, 5000, 50)

plt.figure(figsize=(9, 5))
plt.scatter(ad_budget, sales, c='blue', marker='o', alpha=0.6, label='Data Points')
# 'bo' = blue circle; alpha=0.5 makes points semi-transparent

# Regression line

plt.xlabel("Advertising Budget ($)")
plt.ylabel("Sales Revenue ($)")
plt.legend()
plt.show()
alpha parameter
alpha=0.5 controls transparency of the scatter markers — 0 = fully transparent, 1 = fully opaque. Useful for overlapping data points.
Bar Chart & Histogram
Bar Chart vs Histogram — QB Q2
Bar Chart: Used for categorical data. Each bar represents a discrete category. Gaps between bars. X-axis = categories.
Histogram: Used for continuous/numerical data. Shows frequency distribution. No gaps between bars (continuous). X-axis = value ranges (bins).

Box Plot Anatomy
Box plot shows: Minimum (lower whisker) · Q1 (25th percentile) · Median (Q2, line in box) · Q3 (75th percentile) · Maximum (upper whisker) · Outliers (dots beyond whiskers). IQR = Q3 - Q1. Whiskers extend to 1.5×IQR.

Time Scale Selection Factors (QB Q8)

  • Data granularity: Match time scale to how data is collected (hourly data → daily/hourly scale)
  • Pattern visibility: Daily scale for short-term fluctuations; yearly for long-term trends
  • Business context: Financial reporting uses monthly/quarterly; weather uses daily/hourly
  • Amount of data: Too fine → cluttered; too coarse → hides patterns
  • Seasonality detection: Yearly scale shows annual seasonal patterns

    hue="event",          # QB Q2: hue gives each event a different color
    style="region",        # different line styles per region

    errorbar="sd"          # QB Q5: adds ±1 std dev shading
)

QB Q8 — sizes parameter

PYTHON · DISTRIBUTION PLOTS — HISTOGRAM, KDE, BOXQB Q22

PYTHON · CATEGORICAL PLOTS COMPARISONQB Q24, Q28-Q36
tips = sns.load_dataset("tips")

# STRIPPLOT: categorical scatter, points can overlap
# jitter=True (QB Q28): adds random noise horizontally to separate overlapping points
sns.catplot(data=tips, x="day", y="total_bill", jitter=False)  # QB Q29

# SWARMPLOT: like stripplot but NEVER overlaps (points arranged algorithmically)
# QB Q12: stripplot can overlap; swarmplot cannot → swarmplot shows all points clearly
# dodge (QB Q13): separates hue groups side-by-side in swarmplot
sns.swarmplot(data=tips, x="day", y="tip", hue="sex", dodge=True)

# ORDER parameter (QB Q20): controls display order of categories
sns.catplot(data=tips, x="smoker", y="tip", order=["No", "Yes"])  # QB Q30

# BOXENPLOT (QB Q14): for larger datasets; shows more quantiles than boxplot
# Use boxenplot instead of boxplot when dataset > 1000 rows for more detail
sns.boxenplot(data=tips, x="day", y="total_bill")

# VIOLIN PLOT: combines box plot + KDE; shows full distribution
# SPLIT violin for comparison (QB Q35)
sns.violinplot(data=tips, x="day", y="total_bill", hue="smoker", split=True)

# VIOLIN + SWARM overlay (QB Q31)
g = sns.catplot(data=tips, x="day", y="total_bill", kind="violin", inner=None)
sns.swarmplot(data=tips, x="day", y="total_bill", color="k", size=3, ax=g.ax)

plt.show()

Jointplot vs Pairplot (QB Q4)

Jointplot
Shows relationship between 2 specific variables. Has a central scatter/hex plot with marginal distributions (histogram/KDE) on x and y axes. sns.jointplot(data=df, x='a', y='b', kind='scatter')
Pairplot
Shows relationships between ALL numerical pairs in a dataset. Grid of scatter plots off-diagonal, distributions on diagonal. sns.pairplot(df, hue='species')

QB Q21 — Measure vs Calculated Column
Measure: Computed dynamically at query time based on filter context. Stored as formula, not data. (e.g., Total Sales = SUM(Sales[Amount]))
Calculated Column: Computed at data refresh, stored as actual column in table. Row-by-row calculation. (e.g., Profit = Sales[Revenue] - Sales[Cost])
DAX · ESSENTIAL FUNCTIONSQB Q11, Q12, Q22, Q29, Q30, Q37, Q39
// QB Q22: Average Sales
Average Sales = AVERAGE(Sales[Amount])

// QB Q12: COUNT vs COUNTROWS
// COUNT(column) → counts non-blank values in a column
// COUNTROWS(table) → counts total rows in a table (including blanks)
Count Orders = COUNTROWS(Orders)
Count Products = COUNT(Products[ProductID])

// QB Q30: SUM vs SUMX
// SUM(column) → simple sum of an existing column
Total Revenue = SUM(Sales[Revenue])
// SUMX(table, expression) → iterates each row, evaluates expression, sums results
Total Profit = SUMX(Sales, Sales[Revenue] - Sales[Cost])

// QB Q11: YTD (Year-To-Date)
YTD Sales = TOTALYTD(SUM(Sales[Amount]), Dates[Date])

// QB Q37: Time Intelligence
// MTD (Month-To-Date)
MTD Sales = TOTALMTD(SUM(Sales[Amount]), Dates[Date])

// SAMEPERIODLASTYEAR
Last Year Sales = CALCULATE(SUM(Sales[Amount]),
                    SAMEPERIODLASTYEAR(Dates[Date]))

// QB Q29: Month-over-Month Growth
MoM Growth = 
VAR CurrentMonth = SUM(Sales[Amount])
VAR PreviousMonth = CALCULATE(SUM(Sales[Amount]),
                        DATEADD(Dates[Date], -1, MONTH))
RETURN DIVIDE(CurrentMonth - PreviousMonth, PreviousMonth, 0)

// QB Q39: CALCULATE and FILTER
// CALCULATE: evaluates expression in a modified filter context
High Value Sales = CALCULATE(SUM(Sales[Amount]),
                    Sales[Amount] > 1000)

// FILTER: returns a table filtered by a condition (used inside CALCULATE)
Top Sales = CALCULATE(SUM(Sales[Amount]),
              FILTER(Sales, Sales[Region] = "North"))

Dimensions vs Metrics (QB Q5)

Dimensions
Qualitative attributes used to group data. Green in Looker Studio.
Examples: Country, Product Name, Date, Category
Used for: X-axis, group labels, filters
Metrics
Quantitative values to measure. Blue in Looker Studio.
Examples: Revenue, Sessions, Units Sold, Click Rate
Used for: Y-axis values, KPI cards, aggregations

Editing Schema (QB Q4)

When you edit the schema of a connected Google Sheet in Looker Studio: You can rename fields, change data types (Text/Number/Date), add calculated fields, and set default aggregations. Changes affect how Looker Studio interprets and displays the data but do NOT change the underlying Google Sheet.

LookML (QB Q from course file)

LookML (Looker Modeling Language) is a SQL-based language used in Looker (not Looker Studio) to define data models. It describes:

  • Views: Map to database tables or derived tables
  • Dimensions: Column definitions and transformations
  • Measures: Aggregations (SUM, COUNT, AVERAGE)
  • Explores: Defines which views can be joined together

Building Dashboards in Looker Studio

Building Queries in Looker (Course Syllabus Topics)

  • Filtering: Add dimension filters (e.g., Country = "India") or metric filters (e.g., Revenue > 10000)

Creating Dashboards

  1. Open Looker Studio → Blank Report or from template
  2. Add Data Source → Connect to your data
  3. Insert charts using the toolbar (Add a chart)
  4. Configure each chart: set dimensions, metrics, filters, sort order
  5. Add filter controls for interactivity
  6. Arrange visuals using drag-and-drop
  7. Add drill-downs: right-click chart → Add drill-down
  8. Add annotations: Insert → Text or Shape for context
  9. Share: File → Share → Add people OR Manage access → Get link

Dynamic Content

Use Dimension Control to let viewers switch what dimension is displayed. Data Control lets them switch data sources entirely.
